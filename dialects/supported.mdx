SQLFluff supports 28 SQL dialects. Pass the value in the **`--dialect` flag** column to the CLI or set it in your `.sqlfluff` config file.

```bash
sqlfluff lint query.sql --dialect <flag-value>
```

```ini .sqlfluff
[sqlfluff]
dialect = <flag-value>
```

<Note>
  Dialect support is community-driven and may not cover every feature of a given platform. If you encounter a parse error on valid SQL, the dialect may not yet support that syntax.

  Raise an issue or open a pull request on [GitHub](https://github.com/sqlfluff/sqlfluff/issues) — contributions that add missing syntax are especially welcome.
</Note>

## Checking available dialects

To print the dialects available in your installed version of SQLFluff, run:

```bash
sqlfluff dialects
```

This is useful when you are unsure of the exact flag value, or when running a version of SQLFluff that may differ from this documentation.

## ANSI as the default

If you do not specify a dialect, SQLFluff defaults to `ansi`. ANSI SQL is the base dialect from which all others inherit. It models the SQL standard and is the most conservative option — it does not include vendor-specific extensions.

<Tip>
  Even if your SQL is for a specific platform, setting the correct dialect (e.g. `snowflake`, `bigquery`) gives SQLFluff an accurate grammar to parse against and produces more relevant lint results. Using `ansi` for platform-specific SQL may produce false-positive parse errors.
</Tip>

## Dialect inheritance

Dialects inherit from one another to share common grammar and avoid duplication. The inheritance chain always traces back to `ansi`:

<AccordionGroup>
  <Accordion title="PostgreSQL family">
    `ansi` → `postgres` → `redshift`

    `ansi` → `postgres` → `greenplum`
  </Accordion>
  <Accordion title="MySQL family">
    `ansi` → `mysql` → `mariadb`
  </Accordion>
  <Accordion title="Spark / Databricks family">
    `ansi` → `hive` → `sparksql` → `databricks`
  </Accordion>
</AccordionGroup>

Inheritance is a code-reuse mechanism — it does not guarantee SQL compatibility between a dialect and its parent. See the [dialects overview](/dialects/overview) for a full explanation.
